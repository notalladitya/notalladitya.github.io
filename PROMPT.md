# Portfolio Website Prompt

Create a **software engineer's portfolio website** with the following specifications:

## Design Direction
- `SKILL.md` - Design principles and aesthetic guidelines

## Animations & Interactions
- **Great animations** throughout - smooth, bouncy spring physics
- Scroll-triggered section reveals with staggered animations
- Floating decorative geometric shapes in the background
- Hover effects on cards, buttons, and interactive elements
- Animated progress bars for skills
- Micro-interactions that delight (pulsing elements, wave emoji, etc.)

## Technical Stack
- **Package Manager** pnpm, pnpx for high performance
- **Next.js** (App Router) for static site generation
- **Framer Motion** (or Motion) for smooth, spring-physics animations
- **Tailwind CSS** for styling with custom design tokens
- Static export enabled for easy deployment to GitHub Pages, Netlify, or Vercel
- TypeScript for type safety
- Lucide React for icons

## Sections Required
1. **Hero** - Profile photo with animated rings, greeting with wave emoji, name, title, tech stack badges, CTA buttons
2. **About** - Personal bio, current role highlight, fun fact cards with icons
3. **Skills** - Categorized skill cards (Backend, Frontend, Database, DevOps) with animated progress bars
4. **Experience** - Timeline visualization with color-coded companies, achievements, tech stack per role
5. **Contact** - Social links (Email, LinkedIn, GitHub) with hover effects, playful farewell message
6. **Navigation** - Sticky header with glassmorphism, active section indicator, mobile menu
7. **Footer** - Copyright, "Made with ❤️" message, back to top button

## Design Elements
- Gradient mesh backgrounds
- Subtle noise/grain texture overlay
- Floating circles, squares, triangles as decorations
- Color-coded categories and timeline dots
- Card-based layouts with gentle shadows
- Responsive design for all screen sizes

1. **Not over-the-top professional** - approachable and friendly
2. **Memorable** - distinctive design that stands out
3. **Performant** - static export, optimized animations
4. **Accessible** - proper semantic HTML, readable contrast
5. **Mobile-first** - responsive across all devices
